# **App Name**: OceanWatch

## Core Features:

- Hazard Reporting: Users can submit geotagged reports with photos and videos of observed ocean hazards.
- Map Visualization: Display crowdsourced reports on an interactive map with dynamic heatmaps.
- Social Media Monitoring: Scrape and analyze social media feeds for hazard-related discussions.
- Hotspot Generation: Dynamically generate hotspots based on report volume, keyword frequency, and verified incidents, to quickly identify areas needing more attention. The AI tool reasons over collected data from the tool above to identify the best spots to focus on, including combining real-time reporting with sentiment on social media.
- Role-Based Access: Implement different access levels for citizens, volunteers and disaster managers.
- Filtering and Sorting: Implement various mechanisms for filtering and sorting existing and new data on ocean hazards. Provide functionality for sorting based on geographical proximity to a given location.
- Multilingual support: Cater to different regions and locales with built-in support for various languages.

## Style Guidelines:

- Primary color: HSL(210, 60%, 50%) or RGB Hex code #4A90E2 to convey trust and reliability, echoing the ocean's vastness.
- Background color: HSL(210, 20%, 95%) or RGB Hex code #F0F8FF - a very light tint of blue to keep consistent.
- Accent color: HSL(180, 40%, 60%) or RGB Hex code #77D8D8 for interactive elements, providing a contrast but still consistent with the palette's feel.
- Body and headline font: 'PT Sans', sans-serif, for a modern and clean look.
- Use flat, outlined icons for clarity and visual consistency.
- Implement a responsive grid layout to ensure compatibility across devices.
- Use subtle transitions and animations to enhance user experience without being intrusive.
